# OpenSpec Boundary

## Module-managed

- `openspec/README.md`
- `openspec/templates/`
- placeholder directories and `.gitkeep` files used to keep `openspec/changes/` and `openspec/specs/` present

## Project-owned

- every real change folder under `openspec/changes/`
- every repository-specific spec file under `openspec/specs/`

## Rule

Use the module to distribute scaffold and templates.

Do not use the module as the source of truth for live project changes.

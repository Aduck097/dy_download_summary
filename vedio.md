# AI 视频生成系统需求说明（文生图方案）

## 1. 项目目标

开发一个 **自动视频生成系统**，输入一段文本内容，系统自动生成对应的视频。

视频生成流程：

```
文本脚本
↓
自动拆分分镜
↓
生成图片（文生图）
↓
图片添加简单动画
↓
合成视频
↓
配音 + 字幕（可选）
↓
输出最终视频
```

文生图模型使用：

**Qwen-Image-2.0-Pro（阿里）**

---

# 2. 输入

系统输入内容：

```
一段文本脚本
```

示例：

```
人工智能正在改变世界，
从自动驾驶到医疗诊断，
AI正在进入人类生活的每一个角落...
```

文本长度：

* 100字
* 300字
* 500字
* 1000字

系统需要自动适配。

---

# 3. 分镜生成

系统需要自动把文本拆分为 **视频分镜（Scene）**。

规则：

* 每个分镜时长： **4~6秒**
* 2分钟视频 ≈ **20~30个分镜**

示例输出：

```
Scene 1:
旁白：人工智能正在改变世界
画面描述：未来城市中充满科技感的人工智能场景

Scene 2:
旁白：自动驾驶汽车正在普及
画面描述：无人驾驶汽车在城市道路行驶

Scene 3:
旁白：AI进入医疗行业
画面描述：医生使用人工智能辅助诊断
```

输出结构建议：

JSON

```
[
 {
   "scene_id": 1,
   "narration": "...",
   "image_prompt": "...",
   "duration": 5
 }
]
```

---

# 4. 文生图

每个分镜需要生成一张图片。

使用模型：

**Qwen-Image-2.0-Pro**

图片生成要求：

* 分辨率：

    * 1920x1080（推荐）
* 风格统一
* 画面细节清晰
* cinematic / realistic style

示例 Prompt：

```
futuristic city with artificial intelligence technology,
cinematic lighting,
high detail,
realistic,
8k
```

生成数量：

* 每个分镜 1 张图
* 2分钟视频 ≈ 25张图片

---

# 5. 图片动画

每张图片需要添加 **简单镜头动画**，让画面具有动态效果。

支持动画：

### 1 放大（Zoom In）

```
scale: 1.0 → 1.1
duration: 5s
```

### 2 缩小（Zoom Out）

```
scale: 1.1 → 1.0
```

### 3 横向移动（Pan）

```
left → right
```

### 4 轻微旋转（可选）

```
rotate: 0° → 2°
```

动画效果类似：

**Ken Burns Effect**

---

# 6. 视频合成

系统需要将图片和动画合成为视频。

参数：

```
fps: 30
resolution: 1920x1080
format: mp4
codec: h264
```

视频生成方式建议：

使用：

**FFmpeg**

示例逻辑：

```
图片 → 动画 → 短视频片段
多个片段 → 拼接
输出完整视频
```

---

# 7. 配音（可选）

系统可根据分镜旁白生成配音。

流程：

```
narration
↓
TTS
↓
生成音频
↓
合成到视频
```

TTS可选：

* 阿里语音
* Azure TTS
* Edge TTS

---

# 8. 字幕（可选）

根据旁白自动生成字幕。

字幕格式：

```
SRT
```

示例：

```
1
00:00:00,000 --> 00:00:05,000
人工智能正在改变世界
```
